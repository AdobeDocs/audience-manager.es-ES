# Fuentes de datos privadas {#private-data-feeds}

Un fuente de datos privado es una opción que permite a los proveedores limitar el acceso comprador a sus datos. Los proveedores y compradores de datos deben revisar esta información antes de crear y suscribirse a fuentes de datos privadas.

<!-- c_marketplace_privatefeed.xml -->

## Fuentes de datos privadas para proveedores {#private-data-feeds-providers}

Como proveedor, las fuentes de datos pueden ser públicas o privadas. Un fuente de datos privado le permite limitar el acceso comprador a sus datos, incluido el nombre del vendedor de datos. Es posible que desee crear un fuente de datos privado para oferta ofertas especiales, descuentos o cuando la privacidad y la control de acceso son importantes. Con un fuente de datos privado, puede revisar y aprobar comprador solicitudes. Después de aprobar un solicitud, el fuente se ve igual gustar un fuente de datos público para el comprador. Puede vista y administrar todas las fuentes en **[!UICONTROL Audience Marketplace > My Shared Data]**. Como se muestra a continuación, este tipo de fuente está marcado como &quot;Privado&quot; en la columna de estado.

![](assets/my_shared_data.png)

### Administración de solicitudes de fuentes

Si hace clic en el nombre de una fuente de datos [!UICONTROL My Shared Data] privada, accederá a una Página que contiene varias fichas. Haga clic en un pestaña para administrar sus solicitudes de fuente de datos privado.

## Fuentes de datos privadas para compradores {#private-data-feeds-for-buyers}

Sin embargo, en este caso, la lista de fuente no muestra información resumida sobre características, usuarios únicos y superposición de usuario. Además, el vendedor de datos tiene la opción de mostrar o ocultar su nombre en la [!UICONTROL Provider] columna del [!UICONTROL Marketplace] lista. ## Cómo configurar la relación de uso compartido entre el proveedor de datos y el comprador de datos {#set-up-sharing-relationship}

### Paso 1: Habilitación: proveedor de datos y comprador de datos

El primer paso en el proceso requiere la intervención de Adobe Systems Consultoría o Atención al Cliente. El proveedor de datos y los comprador datos deben ponerse en contacto con Adobe Systems consultor o con el Servicio de atención al cliente para solicitud la habilitación.

### Paso 2: Proveedor de datos: Crear Nuevo Origen de datos

En su Audience Manager cuenta, cree una nueva cookie fuente de datos con:

* **Audience Manager ID** como clave de entrada;
* La **opción Compartir habilitado** está seleccionada.

![](assets/create-datasource.png)

Después de hacer clic en **Guardar**, se crea automáticamente una nueva subcarpeta en **Almacenamiento de características > datos** de terceros.

![](assets/folder-structure.png)

### Paso 3: Proveedor de datos - Identificación de características para compartir

En este paso, identificará las características que desea compartir con su socio. Puede crear nuevos rasgos o editar los existentes. En cualquier caso, necesita las características:

* Para asociarse con la fuente de datos que creó como parte del paso 2.
* Para ser almacenado en la subcarpeta recién creada, bajo datos de terceros.

Obtenga más información sobre la [creación y ](/help/using/features/traits/create-onboarded-rule-based-traits.md)edición de rasgos[](/help/using/features/traits/manage-trait-rules.md#edit-trait).

### Paso 4: Proveedor de datos: Crear fuente de datos

Siguiente, cree una fuente de datos para compartir sus características con los comprador de datos. Consulte [Crear una fuente](/help/using/features/audience-marketplace/marketplace-data-providers/marketplace-create-manage-feeds.md) de datos pública o privada para obtener instrucciones sobre cómo crear una fuente de datos.

>[!IMPORTANT]
>
>En Configuración, seleccione la opción Privado. Si establece este campo como Público, cualquier cliente Audience Marketplace puede suscribirse a su fuente.

![](assets/create-data-feed.png)

### Paso 5: Comprador de datos - Solicitud de acceso

Vaya a **Audience Marketplace > Marketplace**. Search para el fuente de datos creado por el proveedor de datos en el paso anterior. Haga clic en **Solicitar acceso**. El contacto designado del lado del proveedor de datos ahora recibirá un correo electrónico notificación. Consulte también Suscribirse [a una fuente](/help/using/features/audience-marketplace/marketplace-data-buyers/marketplace-manage-subscriptions.md#subscript-private-data-feed) de datos privada.

### Paso 6: Proveedor de datos - Concesión de acceso

Vaya a **Audience Marketplace > Mis datos** compartidos y búsqueda del fuente que creó en el paso 4. Haga clic en el nuevo solicitud de acceso y haga clic en **Permitir acceso** para aprobar el solicitud. Consulte también [Revisar, aprobar o rechazar solicitudes](/help/using/features/audience-marketplace/marketplace-data-providers/marketplace-create-manage-feeds.md#manage-private-requests) de fuentes privadas.

### Paso 7: Comprador de datos - Activar la suscripción

Una vez que el proveedor de datos otorgue acceso al fuente de datos, podrá ver los fuente de su cuenta en **Audience Marketplace > Marketplace**. Revise los detalles, active el botón de suscripción y haga clic en **Revisar y Suscribirse**. Consulte [Almacenamiento para fuentes de datos suscritas](/help/using/features/audience-marketplace/marketplace-data-buyers/marketplace-manage-subscriptions.md#find-subscribed-data-fee) para obtener información sobre dónde encontrar las características de terceros.

Tenga en cuenta que estas características solo se pueden editar en el cuenta del proveedor de datos.
